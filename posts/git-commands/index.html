자주쓰는 Git 명령어

September 18, 2023 · 2 min · 262 words

설정#

이름, 이메일 전역으로 설정#

git git config --global user.name <name>
git git config --global user.email <email>

이름, 이메일 저장소별 설정#

저장소에서 –global 옵션 없이 설정하면 해당 저장소 선언한 이름, 이메일로 설정됨

git git config user.name <name>
git git config user.email <email>

브랜치#

브랜치 생성#

git branch <branch-name>

로컬 브랜치 삭제#

✔현재 작업중인 브랜치는 삭제 불가능

-d는 --delete의 약자

git branch -d <branch-name>

커밋 기록이 실수로 손실되는 걸 방지하기 위해 Git에서 병합되지 않은 변경사항이나 Push 되지 않은 커밋이 존재할 경우 -d 플래그로 브랜치를 삭제할 수 없음 그런 경우에는 -D 플래그를 사용하여 삭제

-D는 –delete –force의 약자( 병합 여부와 관계없이 바로 삭제하기 때문에 주의 )

git branch -D <branch-name>

원격 브랜치 삭제#

원격 브랜치는 push를 통해 삭제

git push <remote-alias> -d <remote-branch-name>

<remote-alias> 저장소 별칭으로 보통은 origin으로 사용

git branch -D <branch-name>

브랜치 목록 조회#

# 로컬 브랜치 목록 조회
git branch

# 원격 브랜치 목록 조회
git branch -r

# 모든 브랜치 목록 조회
git branch -a

# 모든 브랜치 목록 및 각 브랜치 정보 출력
git branch -v

브랜치 이동#

git checkout <branch-name>

# 브랜치를 생성하고 생성한 브랜치로 이동
git checkout -b <branch_name>

브랜치 이름 변경#

git branch -m <old-name> <new-name>

브랜치 병합#

git merge <branch-name>

로컬 작업 내용 임시저장#

현재 상태를 의미하는 임시 메시지가 자동으로 생성됨#

git stash
git stash save

메시지를 직접 등록#

git stash -m <message>

임시 저장된 목록 확인#

git stash list

가장 마지막에 저장된 내용으로 적용#

git stash apply

지정된 내용으로 적용#

stash@{0} ~ stash@{5} 스택 형태로 쌓임

git stash apply
git stash apply stash@{0}

임시 저장된 내용 삭제#

git stash pop
git stash pop stash@{0}
